Charter

HAINPRO is the protocol atlas and protocol program portal within the HAIN ecosystem. Its purpose is to map the emerging protocol landscape, clarify boundaries, identify the missing layer for hybrid AI-native systems, and provide a disciplined path for defining HAIN Protocol (HAINP).

Mission

To organize, compare, and extend the protocol ecosystem relevant to hybrid AI-native systems, and to define the execution and governance semantics that remain underspecified across local, edge, private, and cloud environments.

Scope

  • Map protocol layers across web foundations, tools, agents, interaction, identity, observability, and governance
  • Compare existing protocols by role, boundary, and interoperability assumptions
  • Identify protocol gaps for hybrid execution and policy-aware coordination
  • Define HAINP as the missing execution and governance semantics layer
  • Develop vertical profiles and capability tracks to validate real-world applicability

Non-goals

  • Not a replacement for HTTP or core web standards
  • Not a replacement for MCP as a tool and context access protocol
  • Not a replacement for A2A as a generic agent collaboration protocol
  • Not a low-level transport protocol
  • Not a base automotive, V2X, or robotics control protocol

Protocol object

The core protocol object behind HAINP is the Hybrid Execution Contract: a structured way to express how a task is constrained, routed, executed, degraded, and audited across multiple execution domains.

Initial core concerns

  • Execution domains
  • Locality and residency constraints
  • Policy-aware routing
  • Governed fallback
  • Context portability
  • Trace and audit continuity

Initial priorities

HAINPRO starts as an atlas-first initiative. It begins by structuring the existing protocol ecosystem before expanding into HAINP drafts and profile-based validation.

  • Phase 1: Atlas, comparison, and gap mapping
  • Phase 2: HAINP draft set
  • Phase 3: Profile definition, starting with HAINAUTO
  • Phase 4: Reference patterns, schemas, and implementation guidance

Position within HAIN

  • HAIN World introduces the system
  • HAINOA explains the architecture
  • HAINPRO maps and specifies the protocol layer
  • HAINIZE operationalizes the system into adoption and productization

Compare Protocols

A structured comparison of the emerging protocol stack by scope, actors, and runtime role.

Protocol Primary role Main actors What it helps solve What it does not fully define
MCP Tool and context access AI app, model, tool, external system Connects models and applications to tools and data sources Cross-domain execution semantics, policy-aware routing, governed fallback
A2A Agent collaboration Agent to agent Supports communication and coordination across agents Locality policy, execution-domain semantics, shared fallback contracts
ACP Editor-agent surface IDE/editor, coding agent Standardizes editor-facing agent communication General hybrid execution semantics across environments
HAINP Hybrid execution and governance Nodes, agents, runtimes, policy layers Defines execution domains, locality, routing, fallback, portability, and audit continuity Does not try to replace transport, tool access, or generic agent collaboration layers

Editorial note

Not every protocol competes with every other protocol. Many of the most important protocols are complementary and operate at different layers.

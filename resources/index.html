Initial glossary seeds

  • Execution Domain — the environment in which a task runs, such as local, edge, private, or public.
  • Locality — where data, inference, and context are allowed to remain or move.
  • Routing Intent — the declared preference or requirement that influences where execution should happen.
  • Fallback — the governed action taken when preferred execution is not possible.
  • Trust Zone — a boundary of identity, control, and policy assumptions.
  • Context Portability — the controlled movement of usable context across domains.
